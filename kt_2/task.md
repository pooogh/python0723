### Уровень 1: Базовый синтаксис и наработка памяти

## 1 задача

Вам необходимо создать функцию, которая имитирует процесс сбора данных пользователя при регистрации. В данной задаче мы собираем обязательные поля и любые дополнительные настройки профиля в единый объект. Для этого ваша функция должна принимать базовые данные и произвольное количество именованных аргументов, объединяя их.

**Условия**

- Функция должна возвращать единый словарь.
- В итоговом словаре обязательно должны присутствовать ключи ```username``` и ```email```.
- Все переданные дополнительные именованные аргументы должны быть добавлены в этот же словарь.

**Функции и аргументы**

- Функция ```create_profile()```, которая выполняет сборку данных
- Обязательные позиционные аргументы ```username``` и ```email```
- Аргумент для упаковки произвольного числа именованных параметров (используйте ```**```)

```python
profile_1 = create_profile("admin", "admin@mail.com", role="superuser", is_active=True)
print(profile_1) 
# {'username': 'admin', 'email': 'admin@mail.com', 'role': 'superuser', 'is_active': True}

profile_2 = create_profile("guest", "guest@mail.com", age=25)
print(profile_2) 
# {'username': 'guest', 'email': 'guest@mail.com', 'age': 25}
```

После вызова функции ```create_profile()```, вы получите готовый словарь, который можно использовать для дальнейшей работы с базой данных.

### Уровень 3: Архитектура и паттерны (Хард)

## 7 задача

Вам необходимо создать конвейер обработки данных (middleware). В веб-разработке входящий запрос часто проходит через цепочку обработчиков: очистку, валидацию, форматирование. В данной задаче мы реализуем такой пайплайн с помощью сворачивания списка функций.

**Условия**

- Функция принимает исходную строку и произвольное количество функций-обработчиков.
- Исходная строка должна быть последовательно передана через все функции (результат первой функции становится входом для второй и т.д.).
- Для последовательного применения функций необходимо использовать ```reduce```.

**Функции и аргументы**

- Функция ```apply_pipeline()```, управляющая конвейером
- Аргумент ```data``` (исходная "грязная" строка)
- Упакованный позиционный аргумент ```*processors```, содержащий функции-обработчики

```python
from functools import reduce

# Лямбды-обработчики
strip_spaces = lambda s: s.strip()
to_upper = lambda s: s.upper()
hide_drop = lambda s: s.replace("DROP", "****")

raw_data = "   drop table users; hello!   "

result = apply_pipeline(raw_data, strip_spaces, to_upper, hide_drop)

print(result) # "**** TABLE USERS; HELLO!"
```

После завершения работы ```apply_pipeline()```, исходная строка будет преобразована согласно логике всех переданных обработчиков строго по цепочке.

## 8 задача

Вам необходимо реализовать прототип функции авторизации. В данной задаче мы подготавливаемся к концепции декораторов, проверяя состояние пользователя перед тем, как разрешить выполнение основного кода.

**Условия**

- Функция проверяет словарь запроса ```request_data``` на наличие ключа ```is_authenticated``` со значением ```True```.
- Если проверка пройдена, вызывается и возвращается результат переданной целевой функции ```func```.
- Если проверка не пройдена, возвращается строка ```"403 Forbidden"```, а функция ```func``` не вызывается.

**Функции и аргументы**

- Функция ```require_auth()```, выступающая в роли защитной обертки
- Аргумент ```func``` (целевая функция без аргументов, которую нужно выполнить)
- Аргумент ```request_data``` (словарь с метаданными о текущем пользователе)

```python
def get_secret_data():
    return "Secret Database Information"

valid_request = {"user_id": 1, "is_authenticated": True}
invalid_request = {"user_id": 2, "is_authenticated": False}

print(require_auth(get_secret_data, valid_request))   # "Secret Database Information"
print(require_auth(get_secret_data, invalid_request)) # "403 Forbidden"
```

После применения функции ```require_auth()```, выстраивается безопасный шлюз, который надежно защищает выполнение целевой логики от неавторизованных запросов.

Вам необходимо разработать механизм динамической маршрутизации. В данной задаче мы имитируем процесс поиска нужного обработчика (View) для входящего URL-запроса, используя инструменты функционального программирования.

**Условия**

- Функция должна искать в списке маршрутов словарь, чей ключ ```path``` совпадает с запрошенным адресом.
- Если маршрут найден, необходимо извлечь функцию-обработчик по ключу ```handler``` и **вызвать её**, вернув результат её работы.
- Если маршрут не найден, возвращается строка ```"404 Not Found"```.
- Поиск осуществляется с помощью функции ```filter()```.

**Функции и аргументы**

- Список ```routes```, содержащий словари с путями и функциями
- Функция ```router()```, осуществляющая диспетчеризацию
- Аргумент ```request_path``` (строка с запрашиваемым URL)

```python
routes = [
    {"path": "/home", "handler": lambda: "Home Page"},
    {"path": "/about", "handler": lambda: "About Us"},
    {"path": "/api", "handler": lambda: "JSON Data"}
]

print(router("/home"))    # "Home Page"
print(router("/api"))     # "JSON Data"
print(router("/profile")) # "404 Not Found"
```

После вызова ```router()```, система автоматически найдет и запустит нужный обработчик, обеспечивая правильную реакцию на различные URL-запросы.
